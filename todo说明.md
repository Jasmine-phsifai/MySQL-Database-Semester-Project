# 复旦学生成绩数据库 TODO 说明

## 0. 目标

做一个 `MySQL + Python 后端 + PyQt6 前端` 的学生成绩数据库系统。

要求：

1. 满足 3NF。
2. 不出现明显插入异常、删除异常、修改异常。
3. 以当前有效数据维护为主，不做重型历史追踪。
4. 所有数据库维护都能通过前端按钮或表格操作完成，不依赖命令行。

## 1. 数据库表 TODO

### 1.1 基础实体

```text
Department(院系)
Major(专业)
Student(学生)
Teacher(教师)
Course(课程)
Semester(学期)
```

字段要求：

1. `Department`：院系代码、名称、办公地点、电话、状态。
2. `Major`：专业代码、名称、所属院系、状态。
3. `Student`：学号、姓名、身份证号、宿舍、地址、电话、生日、性别、年级、当前专业、辅修院系、学位等级、状态。
4. `Teacher`：教师编号、姓名、所属院系、状态。
5. `Course`：课程代码、课程名、说明、学时、学分、学位等级、开课院系、是否荣誉课、是否开放 PNP、状态。
6. `Semester`：学期名、开始日期、结束日期。

### 1.2 开课与选课

```text
CourseOffering(某课程在某学期的一次开课)
OfferingTeacher(开课-教师)
Enrollment(学生当前有效选课)
EnrollmentAction(前两周选退课操作日志)
```

规则：

1. 前两周可以自由选课、退课、反复操作。
2. 两周后不能再普通修改选课。
3. 期中只能退课。
4. 期中退课没有成绩。
5. `EnrollmentAction` 只记录前两周操作过程。
6. `Enrollment` 保存最终有效选课状态。

### 1.3 成绩、等第、绩点

```text
GradePolicy(一次开课的评分规则)
GradeBand(等第区间规则)
Grade(学生当前有效成绩)
PnpRequest(学生 PNP 申请)
```

`GradePolicy` 需要支持：

1. 任课老师为每次开课设置百分制到等第的分数段。
2. 及格线不锁死为 60。
3. `A` 和 `A+` 都是满绩点 `4.0`。
4. 只有荣誉课允许 `A+`。
5. `A` 和 `A+` 合计原则上不超过总人数 `30%`。

`GradeBand` 需要支持：

1. 每个等第有原始百分制分数段。
2. 每个等第有绩点区间。
3. 例：`A-` 对应 `3.7 ~ 3.8`。
4. 同一等第内，按原始分数排名，把学生均匀映射到该等第绩点区间。

`Grade` 需要支持：

1. 保存原始百分制分数。
2. 保存计算后的等第。
3. 保存计算后的绩点。
4. 保存同一开课内排名。
5. 保存是否计入学分。
6. 保存是否计入 GPA。
7. 重修后覆盖当前有效成绩。
8. 不要求保存全部旧成绩。

`PnpRequest` 规则：

1. 只有部分课程开放 PNP。
2. 每名学生每学期最多 PNP 一门课。
3. 每名学生总 PNP 学分最多 `16`。
4. PNP 课程不同课程学分不同，按课程学分累计。
5. P 通过，不计入 GPA。
6. NP 不通过，可重修。
7. PNP 在总 GPA 计算中按 `0` 学分处理。

### 1.4 GPA 计算

```text
StudentGpaView(学生 GPA 查询视图或查询结果)
```

规则：

1. 总 GPA = `sum(课程绩点 * 课程学分) / sum(计入GPA的课程学分)`。
2. PNP 不计入 GPA 分子。
3. PNP 不计入 GPA 分母。
4. 期中退课不计入 GPA。
5. 无有效成绩的选课不计入 GPA。

### 1.5 维护辅助

```text
CourseAlias(课程旧代码)
DepartmentChangeNote(院系变动摘要)
OperationLog(JSON 操作日志)
Credential(系统账户凭据)
```

规则：

1. 课程改代码时保留旧代码别名。
2. 院系关闭、合并、拆分只记摘要，当前归属由实体表维护。
3. 所有后端数据库操作写入 JSON 日志。
4. 前端可以读取日志并查询历史操作。

6. 院系不存在时自动创建院系基础记录。

## 3. 前端 TODO

### 3.6 权限与账户

不使用 MySQL 原生角色作为业务权限系统。

账户表：

```text
Credential(账户, bcrypt密码哈希, 角色, 访客类型, 是否启用, 凭据签名)
```

默认账户：

```text
admin / admin123
```

角色：

1. 管理员。
2. 录入员。
3. 访客。

管理员权限：

1. 新增账户。
2. 禁用账户。
3. 重置所有人的密码。
4. 配置访客类型。
5. 配置访客字段脱敏规则。
6. 执行 SQL。
7. 管理所有数据。

录入员权限：

1. 新增数据。
2. 修改数据。
3. 删除数据。
4. 导入导出数据。
5. 不能修改任何人的密码。
6. 不能修改账户权限。

访客权限：

1. 只读。
2. 可按访客类型限制可见表。
3. 可按访客类型限制可见字段。
4. 敏感字段显示为星号。

安全要求：

1. 密码使用 bcrypt 存储。
2. `Credential` 文件本身具有HMAC签名，防止被直接篡改。
3. 访客脱敏规则由管理员配置。
4. 后端校验权限，不能只靠前端隐藏按钮。
